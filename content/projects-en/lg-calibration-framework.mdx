## Visitor Count Prediction Calibration Framework Design

### Project Overview

<img src="/images/CAL final_result.png" alt="CAL result" style={{ width: "100%", borderRadius: "8px" }} />

Enterprise client visitor/consultation predictions were challenged on accuracy, but GT was extremely scarce — directly applying simple calibration models carried high generalization risk.
After structurally decomposing error causes, I designed a 3-Layer framework that detects operational anomalies via Health Check, evaluates error risk via Risk Assessment, and applies Calibration only to low-risk segments.
Visitors used residual correction while consultations used ratio correction, separating strategies by target type.
Built a quality management system applicable to all stores, improving visitor/consultation prediction accuracy while controlling calibration failure risk.

* Visitor MAPE improved 3%p, Consultation MAPE improved 17%p

## Approach

### 1) Redefined the Problem as Quality Management, Not Simple Accuracy Improvement

<img src="/images/CAL layer.png" alt="CAL layer" style={{ width: "100%", borderRadius: "8px" }} />

Directly applying only a calibration model in an extremely low-GT environment risked worsening generalization performance at some stores.
- Redefined the problem from "improving accuracy with a single calibration model" to "building a quality management system for safely applying calibration."
- Judged that a structure for separately monitoring and controlling error causes — upstream module cumulative errors and store environment differences — was needed first.
- Designed a 3-Layer framework applying Health Check, Risk Assessment, and Calibration sequentially.

This transformed the problem into an operationally viable calibration framework even in GT-scarce environments.

### 2) Health Check Design Based on 7 Modules and 35 Metrics

<img src="/images/CAL health_check.png" alt="CAL health_check" style={{ width: "100%", borderRadius: "8px" }} />

Even when prediction errors grow, diagnosing quality status before calibration is difficult without knowing which module the problem started in.
- Divided the AI pipeline into 7 modules and defined 35 metrics explaining each stage's output quality.
- Designed automatic Green / Yellow / Red status determination by calculating per-metric deviation using recent 4-week median and MAD as baselines.
- Made it function both as preprocessing for calibration application decisions and as a monitoring system detecting operational quality anomalies.

Established a foundation for regularly evaluating quality status across 60 stores and screening risky stores before calibration.

### 3) Controlled Correction Intensity via Risk Damping

<img src="/images/CAL final_result.png" alt="CAL result" style={{ width: "100%", borderRadius: "8px" }} />

High-error stores appear to need calibration most, but in low-GT environments, those stores are actually where calibration is most dangerous.
- Designed Risk Damping logic using Decision Tree-based high-error probability estimation, automatically reducing calibration intensity by probability zone.
- Policy divided so high-probability stores receive almost no correction, mid-range stores get partial application, and low-probability stores get full calibration.
- Controlled correction amount through risk-aware weighting rather than using calibration values directly.

Reduced the risk of calibration worsening performance at high-risk stores while securing improvement effects at low-risk stores.

### 4) Separated Calibration Strategies for Visitors and Consultations

<img src="/images/CAL prev_after.png" alt="CAL prev_after" style={{ width: "100%", borderRadius: "8px" }} />

Visitors and consultations have different error generation structures — treating them with the same calibration approach risked unstable consultation performance.
- Determined that visitors were suitable for direct residual correction between predictions and GT, applying Ridge-based residual calibration.
- For consultations, judged that the `consultation / visitor` ratio's stability was higher than consultation count itself, switching to ratio calibration that predicts consultation conversion ratio and multiplies by calibrated visitor count.
- Compared Ridge, Tree, MLP, and KNN throughout this process, ultimately selecting Ridge for its most stable generalization performance in low-GT environments.

Visitors improved stably while consultations achieved larger improvement margins than the residual approach.
