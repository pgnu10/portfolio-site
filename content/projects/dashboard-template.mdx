## mAsh 대시보드 템플릿 체계 기획

### 프로젝트 개요

<img src="/images/DA needs.png" alt="고객 니즈 분류" style={{ width: "100%", borderRadius: "8px" }} />

고객사마다 유사한 분석 요청이 반복됐지만, 무엇을 대시보드로 표준화할 수 있고 무엇은 사람이 직접 분석해야 하는지에 대한 기준이 없었다.
70+개 가설 검증 요청을 대시보드 가능 / 휴먼 리소스 필요 / 불가로 재분류하고, 파악 vs 분석 프레임을 바탕으로 10개 분석 주제와 7종 템플릿 체계를 설계했다.
반복되는 커스텀 대시보드 요청을 재사용 가능한 제품 구조로 전환했고, 실제 고객사에 템플릿 기반 대시보드를 첫 발행하며 기대치 관리와 기획 효율의 기준을 만들었다.

- 70+개 고객 가설 검증 요청을 자동화 가능성 기준으로 구조화
- 현상 파악과 원인 분석을 구분하는 프레임워크 설계로 제품 범위 명확화
- 10개 분석 주제 x 3개 카테고리 체계 정리

- 산점도+사분면, 퍼널, 트래픽 다이어그램 등 신규 위젯 기획
- 7개 업종 사례를 바탕으로 범용 적용 가능성 검증

## 접근 방식

### 1) 반복 요청을 먼저 자동화 가능성 기준으로 재분류

고객 요청은 많았지만 어떤 요청이 제품 기능으로 표준화될 수 있는지에 대한 기준이 없었다.
- 70+개 가설 검증 요청을 하나씩 검토하며 현재 데이터와 플랫폼 기능 기준으로 대시보드 가능, 휴먼 리소스 필요, 불가로 나눴다.
- 반복 가능성과 자동화 가능성을 가장 중요한 기준으로 하여 선별했다.
- 고객 요구를 그대로 받는 대신 제품 구조로 전환 가능한 요청만 먼저 선별했다.

이를 통해 무엇을 템플릿화할 수 있고 없는지 기준이 생기면서 이후 기획과 리소스 배분의 출발점을 명확히 만들 수 있었다.

### 2) 파악과 분석을 구분해 제품 범위를 정의

고객이 말하는 “분석”에는 단순 시각화로 충분한 질문과 해석과 제안까지 필요한 질문이 함께 섞여 있었다.
- 요청을 단순히 주제별로 나누지 않고 파악과 분석이라는 두 층위로 먼저 구분했다.
- 파악은 대시보드 위젯으로 반복 제공 가능한 질문으로 보고, 분석은 분석가의 해석과 결론이 필요한 영역으로 분리했다.
- 이 구분을 통해 대시보드가 해결할 문제와 별도 분석 리포트가 필요한 문제의 경계를 명확히 했다.

대시보드에 대한 과도한 기대를 줄이고 고객 대응과 내부 리소스 투입 기준을 더 일관되게 가져갈 수 있게 됐다.

### 3) 10개 분석 주제를 7종 템플릿으로 구체화

대시보드 가능 요청을 모아도 그대로는 화면 설계로 이어지기 어려웠고 재사용 가능한 단위로 다시 묶을 필요가 있었다.
- 22개의 대시보드화 가능 요청을 고객 이해 / 공간 분석 / 성과 측정 3개 카테고리 아래 10개 분석 주제로 정리했다.
- 그 후 실제 고객이 바로 사용할 수 있는 화면 단위로 재구성해 매장별 실적 비교, 방문객 동선 확인, 퍼널 분석, 상품 간 연관성 분석 등 5종 템플릿으로 구체화했다.
- 이 과정에서 10개 주제는 분석 프레임으로 유지하고, 화면 레벨에서 중복되는 주제는 통합해 템플릿 수를 줄였다.

고객 요청을 일회성 화면이 아니라 재사용 가능한 분석 전달 체계로 바꿀 수 있었고, 실제 발행 가능한 템플릿 구조를 확보했다.

### 4) 기존 위젯으로 답하기 어려운 질문은 신규 위젯으로 보완

구역 간 이동, 전환, 매장 간 비교처럼 구조적인 질문은 기존 차트만으로는 전달력이 부족했다.
- 매장 간 성과를 2차원으로 비교하는 산점도+사분면, 단계별 이탈을 보여주는 퍼널, 구역 간 이동 흐름을 보여주는 트래픽맵을 신규 위젯으로 기획했다.
- 특히 퍼널과 이동 분석은 단순 단계별 집계보다 구역 간 전이를 함께 보여주는 편이 더 적합하다고 판단해, 매장 데이터를 기준으로 전이 행렬 기반 포맷을 검토했다.
- 위젯을 추가한 것이 아니라 고객 질문에 맞는 정보 구조를 새로 설계했다.

기존 위젯의 한계를 보완하면서 템플릿이 실제 고객 질문에 답할 수 있는 수준으로 구체화됐다.

## 성과: 반복되는 분석 요청을 재구성해 범용 템플릿 체계로 제품화
- 73개 고객 요청을 22 / 16 / 35 구조로 재분류해 제품 범위 기준 마련
- 파악 vs 분석 프레임워크로 대시보드와 분석 리포트의 경계 정리
- 10개 분석 주제와 5종 템플릿 체계 구축
- 신규 위젯 기획을 통해 이동/전환/비교형 질문 대응력 강화
- 7개 업종 사례를 바탕으로 범용 적용 가능성 검증
- Fujifilm, Coza 고객사에 템플릿 기반 대시보드 첫 발행
- 이후 고객 기대치 관리와 초기 기획 논의의 기준으로 활용 가능한 프레임 확보
