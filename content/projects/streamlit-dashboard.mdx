## Streamlit 기반 분석 프로토타이핑 체계 구축

## 프로젝트 개요

<img src="/images/Streamlit_sample.png" alt="Streamlit 기반 분석 프로토타이핑 체계 구축" style={{ width: "100%", borderRadius: "8px" }} />

새로운 분석 아이디어가 생겨도 바로 제품 개발로 넘기기엔 비용이 컸고, 말이나 문서만으로는 실제 효용과 표현 방식을 검증하기 어려웠다.  
전환율 분석, 자유 퍼널, 상권 분석, 이상치 탐지, 마케팅 전후 비교, 기간 비교처럼 반복적으로 등장하는 질문을 Streamlit 프로토타입으로 구현해 분석 로직과 UI를 함께 검증할 수 있는 체계를 만들었다.  
분석 아이디어를 코드와 화면으로 빠르게 실험할 수 있는 공통 기반을 마련했고 일부 프로토타입은 이후 운영 리포트, 후속 프로젝트, 제품 기능 검토의 출발점으로 활용됐다.

### 한눈에 보는 결과
* 제품 반영 전 분석 아이디어를 검증하기 위한 Streamlit 기반 프로토타이핑 체계 구축
* 질문 유형별로 분화된 분석 도구 6종 설계 및 구현
* 자유 퍼널 분석을 위한 2~10단계 경로 탐색 구조 구현
* 엄격/비엄격 모드를 지원하는 커스텀 퍼널 알고리즘 설계
* 서울시 공공데이터 API 2종 연동을 통한 상권 분석 프로토타입 구현
* 이상치 탐지 알고리즘 9종 비교 실험 환경 구축
* 마케팅 전후 비교에 MDE/오즈비/p-value/신뢰구간 기반 해석 구조 반영
* 일부 프로토타입은 KPI 이상치 탐지 운영 체계 및 후속 분석 프로젝트로 연결

## 접근 방식

### 1) 분석 아이디어를 제품 반영 전 직접 검증할 수 있는 환경 구축

새로운 분석 관점이 나와도 바로 제품팀에 개발을 요청하기엔 비용이 크고 문서만으로는 실제 유용성과 표현 방식을 충분히 확인하기 어려웠다.  
- 프로덕션 완성도보다 검증 속도를 우선해 Streamlit을 실험 도구로 선택했다.
- 핵심은 대시보드를 많이 만드는 것이 아니라, 분석 아이디어를 실제로 작동하는 화면으로 빠르게 바꿔 내부 검토와 토론의 기준으로 삼는 것이었다.
- 이를 위해 자주 등장하는 질문 유형을 먼저 정리하고 각 질문에 맞는 계산 구조와 인터랙션을 갖춘 소형 프로토타입으로 구현했다.

분석 아이디어를 말로 설명하는 단계에서 벗어나 코드와 화면으로 빠르게 검증할 수 있는 공통 실험 기반이 생겼다.

### 2) 내부 데이터만으로 설명이 부족한 질문은 외부 데이터와 결합

매장 방문 변화나 상권 특성을 해석할 때 내부 방문 데이터만으로는 원인을 충분히 설명하기 어려운 경우가 많았다.
- 서울시 공공데이터 API를 연동해 생활인구와 상권 데이터를 함께 조회할 수 있는 상권 분석 프로토타입을 만들었다.
- 이를 통해 내부 방문 패턴과 외부 생활권 흐름을 한 화면에서 비교할 수 있도록 구성했고 
- 매장 현상만 보는 것이 아니라 주변 맥락까지 함께 해석할 수 있도록 했다.

내부 지표만으로는 답하기 어려운 질문에 대해, 외부 맥락을 결합한 설명 가능성을 빠르게 검증할 수 있었다.

### 3) 프로토타입을 후속 운영 체계와 기능 검토의 실험장으로 활용

프로토타입이 단발성 데모로 끝나면 실제 조직 자산으로 남기 어렵고 후속 프로젝트와 연결되지 않으면 실험 비용 대비 가치가 낮아진다.
- 이상치 탐지 대시보드에서는 9종 detector를 동일한 화면에서 비교할 수 있게 해 어떤 알고리즘이 어떤 패턴에 적합한지 운영 전 단계에서 검토할 수 있도록 했다.
- 마케팅 전후 비교 대시보드에서는 t-test, MDE, 오즈비 같은 해석 기준을 함께 제공해 단순 수치 비교가 아니라 의사결정에 필요한 판단 구조까지 실험했다.

프로토타입은 일회성 실험을 넘어 후속 시스템 설계와 운영 판단을 위한 검증 환경으로 기능하게 됐다.

## 성과: 분석 아이디어를 빠르게 검증 가능한 인터랙티브 도구로 바꾸고 후속 운영 체계와 기능 검토로 연결
* 분석 아이디어 검증을 위한 공통 프로토타이핑 기반 마련
* 질문 유형별로 분화된 프로토타입 6종 설계 및 구현
* 자유 퍼널용 커스텀 알고리즘으로 복잡한 경로 분석 지원
* 외부 상권 데이터 결합을 통해 해석력 확장 가능성 검증
* 이상치 탐지 알고리즘 비교 환경을 후속 운영 체계로 연결
* 마케팅 전후 비교에서 통계 기반 해석 구조 실험
* 분석가가 직접 UI와 계산 구조를 함께 검증하는 작업 방식 정착
* 프로토타입과 프로덕션의 역할 경계를 구분하는 기준 확보
