## Context and Problem

분석 아이디어가 머릿속에만 있으면 아무 의미가 없다. 팀원에게 설명하기도 어렵고, 고객에게 보여주기도 어렵다. "이런 분석이 가능합니다"라고 말로 하는 것과 실제로 인터랙티브 화면을 보여주는 건 완전히 다른 설득력을 가진다.

mAsh 플랫폼에서 다루고 싶은 분석 주제는 많았다. 매장 방문자의 구매 전환 경로, 사용자 이동 동선을 자유롭게 퍼널로 구성해서 분석하는 것, 매장 내부 데이터와 주변 상권 데이터를 대조하는 것. 그런데 이런 기능을 바로 프로덕션에 넣자고 하면 개발 리소스 논의부터 시작해야 하고, 정작 "이게 쓸모 있는 건지"는 다 만들어봐야 알 수 있는 상황이었다.

Streamlit을 선택한 이유는 단순하다. Python을 쓰는 분석가가 프론트엔드 코드 없이 인터랙티브 대시보드를 만들 수 있는 가장 빠른 방법이었기 때문이다. 속도가 핵심이었다.

## My Role and Scope

단독 개발 3종은 기획부터 데이터 처리, 알고리즘, 시각화까지 전부 혼자 했다. mash-labs 3종은 백엔드 엔지니어가 만든 AbstractReport/DashboardLoader 프레임워크 위에서 분석 로직 설계와 시각화 UX를 담당했다. 백엔드 엔지니어가 `detectors.py`(9종 탐지기 클래스), `preprocessor.py` 등을 구현했고, 나는 어떤 알고리즘을 어떤 조합으로 쓸지, 결과를 어떻게 보여줄지를 설계했다.

## Approach and Design

### 단독 개발: 전환율 분석 (conversion_drives)

매장 방문자가 어떤 존(zone)을 거쳐 어디서 전환되는지를 추적하는 대시보드다. ESG_PREFIX별 상관 계수, 방문 빈도, 전환율 매트릭스를 한 화면에 보여준다.

만들어보고 느낀 건, 전환율이라는 개념 자체가 오프라인 매장에서는 온라인만큼 깔끔하게 정의되지 않는다는 점이다. 웹에서는 클릭 한 번이 명확한 이벤트지만, 오프라인에서는 "이 사람이 제품을 만져봤다"와 "그냥 지나갔다"의 경계가 흐릿하다. 그래서 다양한 상관 관계를 동시에 볼 수 있는 매트릭스 뷰를 채택했다.

### 단독 개발: 자유 퍼널 분석 (free_funnel)

이 대시보드가 기술적으로 가장 도전적이었다.

사용자가 2단계부터 10단계까지 퍼널을 직접 설정할 수 있다. "입구 → A존 → B존 → 퇴장" 같은 경로를 자유롭게 정의하면, 각 단계별 이탈률과 전환률을 산출한다. 엄격 모드(정확히 그 순서대로 이동한 사람만)와 비엄격 모드(중간에 다른 곳을 거쳐도 OK)를 모두 지원한다.

문제는 성능이었다. 단계가 늘어날수록 조합이 폭발적으로 증가한다. 방문객 수천 명의 이동 경로를 10단계 퍼널로 필터링하면, 순진한 구현으로는 느려서 못 쓴다.

> **Decision: 퍼널 연산 최적화 방식**
>
> **검토한 대안:** Pandas iterrows 기반 행 단위 처리 — 구현은 간단하지만 대용량에서 너무 느림
> **선택:** NumPy `user_step_matrix` 기반 하이브리드 최적화 알고리즘 직접 설계
> **근거:** 각 사용자의 이동 경로를 step matrix로 벡터화하면, 퍼널 매칭을 행렬 연산으로 처리할 수 있다. 엄격/비엄격 모드 전환도 matrix 연산 조건만 바꾸면 된다. 라이브러리를 가져다 쓴 게 아니라 문제 구조에 맞게 직접 설계한 알고리즘이다.

### 단독 개발: 상권 분석 (market_analysis)

매장 내부 데이터만으로는 한계가 있다. "이 매장 앞을 하루에 몇 명이 지나가는가", "이 동네 업종별 매출 추이는 어떤가" — 이런 외부 컨텍스트가 있어야 내부 데이터의 의미가 풍부해진다.

서울시 생활인구 API와 서울시 상권 매출 API, 두 가지 공공데이터를 연동했다. 업종별 성별/연령대/요일/시간대별 판매 비율과 매장 전환율을 크로스로 분석할 수 있게 구현했다. API 호출 제한과 데이터 갱신 주기가 제약이긴 했지만, "외부 데이터와 내부 데이터를 같이 보면 이런 인사이트가 나온다"는 걸 증명하기에는 충분했다.

### mash-labs 협업: 이상치 탐지 (anomaly_detection)

9종 탐지 알고리즘을 하나의 대시보드에서 인터랙티브하게 적용할 수 있게 설계했다.

| 알고리즘 | 특성 |
|----------|------|
| IQR | 사분위 기반, 분포 가정 없음 |
| ZScore | 정규분포 가정, 빠른 계산 |
| MAD | Median 기반, 아웃라이어에 강건 |
| Hampel | 이동 윈도우 기반 MAD |
| Bollinger | 이동 평균 + 표준편차 밴드 |
| Weekday | 요일별 패턴 기반 |
| YoY | 전년 동기 대비 |
| 비율 기반 | 지표 간 비율 변화 감지 |
| Custom Threshold | 사용자 정의 임계값 |

왜 9종이나 넣었는가? 하나의 알고리즘으로는 모든 종류의 이상치를 잡을 수 없기 때문이다. 계절성이 강한 매장은 YoY가 유용하고, 급격한 변화를 잡으려면 ZScore가 낫고, 아웃라이어가 많은 데이터에서는 MAD가 안정적이다. 사용자가 데이터 특성에 맞는 알고리즘을 직접 선택하고 비교할 수 있게 한 것이 핵심이다. Altair 기반 시각화를 적용했다.

이 대시보드의 탐지 로직은 이후 KPI 이상치 탐지 시스템의 운영 보고서와도 연계되었다.

### mash-labs 협업: 마케팅 전후 비교 (marketing_analytics_periods)

마케팅 캠페인 효과를 "느낌"이 아니라 숫자로 판단하기 위한 대시보드다. Pre/Post 기간을 설정하면 MDE(Minimum Detectable Effect), 신뢰구간, p-value, 오즈비(Odds Ratio)를 산출한다.

통계적 유의성 검정을 넣은 이유는 간단하다. "캠페인 후 방문객이 10% 늘었어요"라고 말하는 것과 "95% 신뢰수준에서 유의미한 증가입니다"라고 말하는 건 의사결정에 미치는 영향이 다르다. SciPy의 통계 검정 모듈을 활용했다.

### mash-labs 협업: 기간 비교 (time_compare)

가장 단순한 대시보드지만 가장 자주 쓰이는 것이기도 하다. 기준 기간과 비교 기간의 일별 추이를 겹쳐서 보여주고, 매장별 합산/평균 비교를 제공한다. "지난달 대비 이번 달 어떤가"라는, 가장 기본적이면서도 가장 빈번한 질문에 답하는 도구다.

## Impact and Outcome

- **6종 대시보드 구축 완료**: 단독 3종 + mash-labs 협업 3종. 분석 아이디어를 제품 반영 전에 빠르게 검증하는 프로세스가 만들어졌다.
- **자유 퍼널 알고리즘**: NumPy 기반 하이브리드 최적화로 10단계 퍼널도 실시간 산출 가능. 단순히 라이브러리를 가져다 쓴 게 아니라 문제에 맞춘 커스텀 알고리즘을 설계한 사례.
- **외부 데이터 연계 검증**: 서울시 API와 내부 데이터를 결합하면 어떤 인사이트가 가능한지 실증. 상권 분석 대시보드가 그 가능성을 보여줬다.
- **이상치 탐지 9종 비교 환경**: 사용자가 데이터 특성에 맞는 알고리즘을 직접 선택하고 결과를 비교할 수 있는 도구. 운영 보고서와 연계.
- **기술 스택 확장**: Observable Framework 조사를 통해 Streamlit 외 대안도 탐색. 다만 프로토타이핑 속도 면에서 Streamlit이 여전히 우세하다는 판단.

프로토타입 수준이라 프로덕션 전환 시에는 추가 작업이 필요하다. Streamlit의 성능 한계(대용량 처리 시 응답 속도)도 실감했다. 그래도 "이게 쓸모 있는 분석인가?"를 먼저 확인하고 개발에 들어가는 흐름을 만든 건 의미가 있다.
